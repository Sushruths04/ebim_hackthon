Phase 1 proof bundle — headless Task 3 episode harness (v0.1-harness)
=====================================================================

Claim
-----
The headless episode runner completes a full idle episode (video +
EPISODE_RESULT JSON + grading of stages 1/3/4), and the same seed twice
produces bit-identical spawn poses (300 beans + 5 grading props).

Evidence in this directory
--------------------------
- episode.mp4          runA video, 160 frames @ 20 fps (8 s sim time)
- result_runA.json     full EPISODE_RESULT of run A
- result_runB.json     full EPISODE_RESULT of run B (independent process)
- spawn_positions and spawn_bean_count are identical across both files
  (diff them, or see SPAWN_MATCH: True in the session log).
  All stage scores are 0 by design: the idle policy never moves the
  robot; Phase 1 verifies the harness, not task behavior.

Reproduce
---------
Machine: GCP VM sim-dev-g4b (g4-standard-48, FULL RTX PRO 6000 96 GB,
spot, us-central1-b), Ubuntu + apt nvidia-driver-580-open, container
isaac-lab-2-3-2-workshop (Isaac Sim 5.1.0), repo bind-mounted at
/workspace/EBiM_Challenge.

Commit: b4be9d5 (branch main). The in-file "git_commit": "unknown" is a
container git-ownership artifact (safe.directory not set inside the
container); the checkout was b4be9d5, pulled immediately before the runs.

Command (run twice, out-dirs task3_det_runA / task3_det_runB):
  python scripts/task3/run_episode.py --seed 42 --head-placement a \
    --policy idle --record-video \
    --out-dir /workspace/EBiM_Challenge/outputs/task3_det_<RUN>

Wall time: 95.8 s (runA) / 95.5 s (runB) per episode including app boot.
Date: 2026-07-17, ~16:07 and ~16:22 UTC.
